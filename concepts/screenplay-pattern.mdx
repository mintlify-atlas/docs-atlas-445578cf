## What is the Screenplay Pattern?

The Screenplay pattern is a design approach that separates test automation concerns into three main components:

1. **Tasks** - Business logic and user actions (the "how")
2. **Questions** - Validations and assertions (the "what to verify")
3. **UI Selectors** - Element locators (the "where")

This separation makes tests more readable, maintainable, and reusable.

## Benefits

- **Readable Tests**: Tests read like user stories
- **Reusable Components**: Share Tasks and Questions across tests
- **Easy Maintenance**: Change selectors in one place
- **Clear Separation**: Business logic separated from technical implementation

## Component Breakdown

### UI Selectors (Target)

UI selectors are centralized element locators organized by page or component. They define **where** elements are located.

**File:** `cypress/support/ui/homeTarget.js`

```javascript
export const JUGAR_GRATIS = 'div[class="top-content"]  a[role="button"]  div[data-testid="cta-content"]';
export const BUTTON_GUARDAR = 'div[role="dialog"] button[class*="type_save"]';
export const IMAGE_POPPY = 'a[aria-label^="Versión 16.6"] img[data-testid="mediaImage"]';
export const ASSASIN_AKALY = 'section[data-testid="blade"] div[class="icon-tab-media-subtitle"]';
export const DIVS_ELEMENT = 'div[class="cta-content"]';
export const NOTICIAS_LIST = '[data-testid$="internal-news"] > p';
export const NOTICIAS_ELEMENT = '[data-testid$="internal-news"]  a[data-testid^="riotbar:desktopNav:link-"]';
```

<Accordion title="Supported Selector Types">
  ```javascript
  // IDs
  export const MY_BUTTON = '#mi-boton';

  // Classes
  export const PRIMARY_BTN = '.btn-primary';

  // Attributes
  export const LOGIN_INPUT = '[data-testid="login"]';

  // Combinations
  export const USER_INPUT = 'form > div input[name="user"]';
  ```
</Accordion>

<Tip>
  Keep selectors organized by page or feature. This makes it easier to update when the UI changes.
</Tip>

### Tasks

Tasks represent user actions or business logic. They define **how** to interact with the application.

**File:** `cypress/support/tasks/homeTask.js`

<CodeGroup>
```javascript Element Count Task
import {CreateTask} from '@utils/imageEvidence'

/**
 * Count the number of elements with the same text value and element.
 *
 * @example
 * ElementCount('#mi-boton', textValue, alias)
 * ElementCount('.btn-primary', textValue, alias)
 * ElementCount('[data-testid="login"]', textValue, alias)
 */
export const ElementCount = (elementSelector, keyWord, alias) => {
    cy.log(`Thinking: encontrar numero de elementos ${elementSelector} que tienen en su texto {${keyWord}} y sea visible...`)
    cy.get(elementSelector) 
      .filter(`:contains(${keyWord})`) 
      .filter(':visible')            
      .its('length')
      .as(alias) 
}
```

```javascript Click Task
/**
 * Click on a web element
 *
 * @example
 * click('#mi-boton', textValue)
 * click('.btn-primary', textValue)
 * click('[data-testid="login"]', textValue)
 */
export const click = (elementSelector, textElement) => {
    cy.log(`Thinking: dar click sobre boton {${textElement}}`)
    cy.get(elementSelector).contains(textElement).click();
}
```

```javascript Show List Task (with Evidence)
import {CreateTask} from '@utils/imageEvidence'

/**
 * Show list of elements linked to webElement
 *
 * @example
 * showList('#mi-boton', textValue)
 * showList('.btn-primary', textValue)
 */
export const showList = CreateTask('Desplegar lista de opciones', (elementSelector, textElement) => {
    cy.log(`Thinking: dar click sobre boton {${textElement}}`)
    cy.get(elementSelector).contains(textElement).realHover();
});
```
</CodeGroup>

<Note>
  Tasks wrapped with `CreateTask()` automatically capture screenshot evidence after execution using the custom `cy.evidence()` command.
</Note>

#### CreateTask Utility

**File:** `cypress/support/utils/imageEvidence.js`

```javascript
export const CreateTask = (nameTask, logicTask) => {
    return (...args) => {
        logicTask(...args);
        cy.evidence(nameTask);        
    };
};
```

The `CreateTask` wrapper:
1. Executes the task logic
2. Automatically captures a screenshot with the provided name
3. Returns a reusable function

### Questions

Questions represent validations and assertions. They answer queries about the application state and define **what** to verify.

**File:** `cypress/support/questions/elementQuest.js`

<CodeGroup>
```javascript Element Existence
import {CreateTask} from '@utils/imageEvidence'

/**
 * Verify the element exists in the DOM.
 *
 * @example
 * ElementExit('#mi-boton')
 * ElementExit('.btn-primary')
 * ElementExit('[data-testid="login"]')
 */
export const ElementExit = CreateTask('Verificar que el elemento exista', (elementSelector) => {
    cy.log(`Thinking: Verificando que el elemento ${elementSelector} ya existe...`);
    return cy.get(elementSelector)
      .scrollIntoView({ block: 'center', inline: 'center' })
      .first()
      .should('exist');
});
```

```javascript Element Visibility
import {CreateTask} from '@utils/imageEvidence'

/**
 * Verify the element is visible in the DOM.
 *
 * @example
 * ElementVisible('#mi-boton')
 * ElementVisible('.btn-primary')
 * ElementVisible(AliasDefinido)
 */
export const ElementVisible = CreateTask('Verificar que el elemento sea visible', (elementSelector) => {
    cy.log(`Thinking: Verificando que el elemento ${elementSelector} es visible ...`);
    return cy.get(elementSelector)
      .scrollIntoView({ block: 'center', inline: 'center' })
      .first()
      .should('be.visible');
});
```

```javascript Text Visibility
import {CreateTask} from '@utils/imageEvidence'

/**
 * Verify the text is visible in the DOM.
 *
 * @example
 * VisibleText('#mi-boton', keyWord)
 * VisibleText('.btn-primary', keyWord)
 * VisibleText('[data-testid="login"]', keyWord)
 */
export const VisibleText = CreateTask('Verificar que el texto sea visible', (elementSelector, keyWord) => {
    cy.log(`Thinking: Verificando que el elemento ${elementSelector} contenga el texto {${keyWord}} y sea visible...`);
    return cy.get(elementSelector)
      .contains(keyWord)
      .first()
      .scrollIntoView({ block: 'center', inline: 'center' })
      .should('be.visible');
});
```

```javascript Image Type Check
import {CreateTask} from '@utils/imageEvidence'

/**
 * Verify the element is an image of a specific type.
 *
 * @example
 * CheckImageType('#mi-boton', 'png')
 * CheckImageType('.btn-primary', 'jpg')
 */
export const CheckImageType = CreateTask('Verificar que el elemento sea una imagen de tipo', (elementSelector, imageType) => {
    cy.log(`Thinking: Verificando que el elemento ${elementSelector} sea una imagen de tipo {${imageType}} y sea visible...`);

    cy.get(elementSelector)
      .should('be.visible') 
      .and('have.attr', 'src') 
      .and('include', imageType);
});
```
</CodeGroup>

<Accordion title="More Question Examples">
  ```javascript
  // Element does not exist
  export const ElementNotExit = (elementSelector, textElement) => {
      cy.log(`Thinking: Verificando que el elemento ${elementSelector} no existe...`);
      return cy.get(elementSelector)
        .contains(textElement)
        .should('not.exist');
  }

  // Element is not visible
  export const ElementNotVisible = (elementSelector, textElement) => {
      cy.log(`Thinking: Verificando que el elemento ${elementSelector} no es visible ...`);
      return cy.get(elementSelector)
        .contains(textElement)
        .should('not.be.visible');
  }

  // Check element count from alias
  export const CheckElementNumber = (Alias, numberElement) => {
      cy.log(`Thinking: Verificando que el numero de elementos visibles sea el correcto ...`)
      cy.get(`@${Alias}`)
        .should('equal', numberElement);
  }
  ```
</Accordion>

## Using the Pattern in Tests

Here's how all three components work together in a test file:

**File:** `cypress/e2e/checkElementHome.cy.js`

```javascript
import * as HomeUI from '@ui/homeTarget'
import * as Quest from '@questions/elementQuest' 
import * as Tasks from '@tasks/homeTask'

const HISTORY = 'HISTORIAS';
const NEWS = 'Noticias';
const FREE_ALIAS = 'gratisElement';
const NOW_ALIAS = 'ahoraElement';

describe('My first Cypress test', () => {
    beforeEach(() => {
        cy.visit('es-es');
    })

    it('search for elements on lol page', () => {
        // Questions - verify element state
        Quest.ElementExit(HomeUI.JUGAR_GRATIS);
        Quest.ElementVisible(HomeUI.JUGAR_GRATIS);
        Quest.CheckImageType(HomeUI.IMAGE_POPPY, 'jpg');
        Quest.VisibleText(HomeUI.ASSASIN_AKALY, 'La Asesina Sigilosa');
        
        // Tasks - perform actions
        Tasks.ElementCount(HomeUI.DIVS_ELEMENT, 'Jugar gratis', FREE_ALIAS);
        Tasks.ElementCount(HomeUI.DIVS_ELEMENT, 'Jugar ahora', NOW_ALIAS);
        
        // Questions - verify counts
        Quest.CheckElementNumber(NOW_ALIAS, 3);
        Quest.CheckElementNumber(FREE_ALIAS, 2);
    });

    it('search for elements in lists', () => {
        Quest.VisibleText(HomeUI.NOTICIAS_LIST, NEWS);
        Quest.ElementNotVisible(HomeUI.NOTICIAS_ELEMENT, HISTORY);
        Quest.ElementNotExit(HomeUI.NOTICIAS_ELEMENT, 'PATRICIO');
        
        // Task - show list
        Tasks.showList(HomeUI.NOTICIAS_LIST, NEWS);
        
        // Question - verify list item is now visible
        Quest.VisibleText(HomeUI.NOTICIAS_ELEMENT, HISTORY);
    });
});
```

## Best Practices

<Accordion title="Keep Tests Readable">
  Write tests that read like user stories. The Screenplay pattern enables this:

  ```javascript
  // Good - reads naturally
  Quest.ElementVisible(HomeUI.LOGIN_BUTTON);
  Tasks.click(HomeUI.LOGIN_BUTTON, 'Login');
  Quest.VisibleText(HomeUI.WELCOME_MESSAGE, 'Welcome back!');

  // Avoid - too technical
  cy.get('[data-testid="login-btn"]').should('be.visible').click();
  cy.get('.welcome').should('contain', 'Welcome back!');
  ```
</Accordion>

<Accordion title="Organize by Domain">
  Create separate files for different pages or features:

  ```
  cypress/support/
  ├── ui/
  │   ├── homeTarget.js
  │   ├── loginTarget.js
  │   └── dashboardTarget.js
  ├── tasks/
  │   ├── homeTask.js
  │   ├── loginTask.js
  │   └── dashboardTask.js
  └── questions/
      ├── homeQuest.js
      ├── loginQuest.js
      └── dashboardQuest.js
  ```
</Accordion>

<Accordion title="Use CreateTask for Evidence">
  Wrap important Tasks and Questions with `CreateTask()` to automatically capture screenshots:

  ```javascript
  import {CreateTask} from '@utils/imageEvidence'

  export const submitForm = CreateTask('Submit registration form', (formData) => {
      // Task logic
      cy.get('#submit').click();
  });
  ```
</Accordion>

<Accordion title="Leverage Webpack Aliases">
  Use webpack aliases for clean imports:

  ```javascript
  // Good - using aliases
  import * as HomeUI from '@ui/homeTarget'
  import * as Quest from '@questions/elementQuest'
  import * as Tasks from '@tasks/homeTask'

  // Avoid - relative paths
  import * as HomeUI from '../../support/ui/homeTarget'
  import * as Quest from '../../support/questions/elementQuest'
  ```

  Learn more about [Webpack Aliases](/concepts/webpack-aliases).
</Accordion>

<Tip>
  The Screenplay pattern scales well. As your test suite grows, you'll appreciate the clean separation and reusability.
</Tip>
