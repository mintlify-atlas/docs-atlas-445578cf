## Overview

The `cy.evidence()` command is a custom Cypress command that captures screenshots with contextual task names. It's implemented in `cypress/support/commands.js` and is automatically called by all validation functions through the `CreateTask` wrapper utility.

## Basic Usage

### Manual Screenshot Capture

You can manually capture evidence screenshots at any point in your test:

```javascript
it('should capture screenshot with context', () => {
  cy.visit('es-es');
  cy.get('[data-testid="card-category"]')
    .contains('Actualizaciones del juego')
    .evidence('Screenshot perfecto y centrado');
});
```

**Output:**
- Console log: `📸 TASK: Screenshot perfecto y centrado`
- Screenshot file: `Task-Screenshot perfecto y centrado.png`

### Chaining with Custom Commands

Combine `evidence()` with other custom commands like `centerElement()`:

```javascript
it('should center and capture element', () => {
  cy.visit('es-es');
  cy.get('[data-testid="card-category"]')
    .contains('Actualizaciones del juego')
    .centerElement()
    .evidence('Screenshot perfecto y centrado');
});
```

<Note>
  The `evidence()` command is chainable, allowing you to continue working with the element after capturing the screenshot.
</Note>

## Implementation Details

### Command Definition

The `cy.evidence()` command is defined in `cypress/support/commands.js`:

```javascript
Cypress.Commands.add('evidence', (nameTask) => {
  Cypress.log({
    name: '📸 TASK',
    message: nameTask,
  });

  cy.screenshot(`Task-${nameTask}`, { 
    capture: 'runner', 
    overwrite: false 
  });
});
```

**Key features:**
- **Custom log entry:** Adds a formatted log entry with 📸 emoji for easy identification
- **Descriptive filename:** Prefixes screenshots with `Task-` followed by the task name
- **Runner capture:** Captures the entire test runner, including command log and test content
- **No overwrite:** Preserves all screenshots, allowing multiple captures with the same name

### Screenshot Configuration

<CodeGroup>
```javascript capture: 'runner'
// Captures entire test runner (default for evidence)
cy.evidence('Full context screenshot');
// Result: Shows test commands, assertions, and viewport
```

```javascript capture: 'viewport'
// For viewport-only screenshots, use cy.screenshot directly
cy.screenshot('Viewport only', { capture: 'viewport' });
// Result: Shows only the application viewport
```

```javascript capture: 'fullPage'
// For full page screenshots, use cy.screenshot directly
cy.screenshot('Full page', { capture: 'fullPage' });
// Result: Captures entire scrollable page
```
</CodeGroup>

## Automatic Evidence with CreateTask

### CreateTask Utility

The `CreateTask` utility (located in `cypress/support/utils/imageEvidence.js`) automatically adds evidence capture to any validation function:

```javascript
export const CreateTask = (nameTask, logicTask) => {
  return (...args) => {
    logicTask(...args);
    cy.evidence(nameTask);        
  };
};
```

### How It Works

1. Wraps a validation function with automatic evidence capture
2. Executes the validation logic
3. Captures screenshot with the task name
4. Returns the result for chaining

**Example from elementQuest.js:**

```javascript
import {CreateTask} from '@utils/imageEvidence'

export const ElementVisible = CreateTask(
  'Verificar que el elemento sea visible',
  (elementSelector) => {
    cy.log(`Thinking: Verificando que el elemento ${elementSelector} es visible ...`);
    return cy.get(elementSelector)
      .scrollIntoView({ block: 'center', inline: 'center' })
      .first()
      .should('be.visible');
  }
);
```

**When called:**
```javascript
Quest.ElementVisible('[data-testid="login"]');
```

**Results in:**
1. Log: `Thinking: Verificando que el elemento [data-testid="login"] es visible ...`
2. Assertion: Element is visible
3. Log: `📸 TASK: Verificar que el elemento sea visible`
4. Screenshot: `Task-Verificar que el elemento sea visible.png`

<Note>
  All validation functions in `elementQuest.js` automatically capture evidence screenshots, eliminating the need to manually add `cy.evidence()` calls.
</Note>

## Real Test Examples

### Example 1: Environment Variables with Evidence

```javascript
const secretValue = Cypress.env('SECRET_VALUE');
const variableRepo = Cypress.env('VARIABLE_REPO');

it(`ejemplo nuevo task creada para cy y varibales  secreta de ambiente ${secretValue} variable normal de repo ${variableRepo}`, () => {
  cy.log(`valor secreto: ${secretValue}`);
  cy.log(`variable repo: ${variableRepo}`);
  cy.get('[data-testid="card-category"]')
    .contains('Actualizaciones del juego')
    .centerElement()
    .evidence('Screenshot perfecto y centrado');
});
```

**Output:**
```
✓ Log: valor secreto: my-secret
✓ Log: variable repo: my-repo-var
✓ Element found and centered
✓ 📸 TASK: Screenshot perfecto y centrado
```

### Example 2: Multiple Evidence Points

```javascript
it('should capture evidence at key validation points', () => {
  cy.visit('es-es');
  
  // Automatic evidence from Quest functions
  Quest.ElementExit(HomeUI.JUGAR_GRATIS);
  // → Screenshot: Task-Verificar que el elemento exista.png
  
  Quest.ElementVisible(HomeUI.JUGAR_GRATIS);
  // → Screenshot: Task-Verificar que el elemento sea visible.png
  
  Quest.CheckImageType(HomeUI.IMAGE_POPPY, 'jpg');
  // → Screenshot: Task-Verificar que el elemento sea una imagen de tipo.png
  
  // Manual evidence for custom verification
  cy.get('.custom-component')
    .should('have.class', 'active')
    .evidence('Custom component is active');
  // → Screenshot: Task-Custom component is active.png
});
```

### Example 3: Evidence in Complex Workflows

```javascript
it('should document entire user flow', () => {
  cy.visit('es-es');
  
  // Step 1: Verify initial state
  Quest.VisibleText(HomeUI.NOTICIAS_LIST, NEWS);
  // → Screenshot captured automatically
  
  // Step 2: Verify hidden elements
  Quest.ElementNotVisible(HomeUI.NOTICIAS_ELEMENT, HISTORY);
  // → Screenshot captured automatically
  
  // Step 3: Perform action with manual evidence
  cy.get(HomeUI.NOTICIAS_LIST)
    .contains(NEWS)
    .click()
    .evidence('Lista expandida después del click');
  
  // Step 4: Verify new state
  Quest.VisibleText(HomeUI.NOTICIAS_ELEMENT, HISTORY);
  // → Screenshot captured automatically
});
```

## Screenshot Organization

### File Naming Convention

All evidence screenshots follow the pattern: `Task-{nameTask}.png`

**Examples:**
- `Task-Verificar que el elemento exista.png`
- `Task-Verificar que el elemento sea visible.png`
- `Task-Verificar que el texto sea visible.png`
- `Task-Screenshot perfecto y centrado.png`
- `Task-Custom component is active.png`

### Screenshot Location

By default, Cypress stores screenshots in:
```
cypress/screenshots/{spec-name}/{screenshot-name}.png
```

**Example structure:**
```
cypress/
  screenshots/
    checkElementHome.cy.js/
      Task-Verificar que el elemento exista.png
      Task-Verificar que el elemento sea visible.png
      Task-Screenshot perfecto y centrado.png
```

<Note>
  With `overwrite: false`, if you run the same test multiple times, Cypress appends numbers to prevent overwriting: `Task-name.png`, `Task-name (1).png`, `Task-name (2).png`
</Note>

## Best Practices

### Use Descriptive Task Names

<CodeGroup>
```javascript Good
cy.evidence('Login form submitted successfully');
cy.evidence('User dashboard loaded with profile data');
cy.evidence('Payment modal displayed after checkout');
```

```javascript Avoid
cy.evidence('Test 1');
cy.evidence('Screenshot');
cy.evidence('Check');
```
</CodeGroup>

### Strategic Evidence Placement

<Note>
  **Capture evidence at critical points:**
  - Before and after important actions
  - After navigation or page transitions
  - When validating complex UI states
  - At the end of multi-step workflows
</Note>

### Combine with Other Commands

```javascript
// Perfect combo: Center element, then capture evidence
cy.get('.hero-banner')
  .centerElement()
  .evidence('Hero banner centered and visible');

// Capture evidence after interaction
cy.get('.submit-button')
  .click()
  .evidence('Form submitted');

// Chain multiple operations
cy.get('.dropdown')
  .select('Option 1')
  .evidence('Dropdown option selected')
  .should('have.value', 'option1');
```

### Avoid Excessive Screenshots

<Note>
  While evidence is valuable, too many screenshots can:
  - Slow down test execution
  - Consume disk space
  - Make it harder to find relevant evidence
  
  Use automatic evidence from validation functions and add manual evidence only for custom verifications or critical workflow steps.
</Note>

## Cypress Log Output

When `cy.evidence()` is called, you'll see formatted output in the Cypress command log:

```
📸 TASK: Screenshot perfecto y centrado
  ↳ screenshot: Task-Screenshot perfecto y centrado.png
```

This makes it easy to:
- Identify when evidence was captured
- See the context/purpose of each screenshot
- Trace test execution flow
- Debug failures with visual context

## Configuration Options

You can customize screenshot behavior in `cypress.config.js`:

```javascript
module.exports = {
  screenshotsFolder: 'cypress/screenshots',
  screenshotOnRunFailure: true,
  trashAssetsBeforeRuns: true,
  video: true,
  videoCompression: 32,
};
```

<Note>
  The `cy.evidence()` command respects your Cypress screenshot configuration but uses hardcoded `capture: 'runner'` and `overwrite: false` for consistency.
</Note>
