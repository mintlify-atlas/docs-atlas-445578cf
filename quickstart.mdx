# Quickstart Guide

This guide will help you run your first test and understand the basics of working with this Cypress E2E automation framework.

## Run Your First Test

<Steps>
  <Step title="Open Cypress Test Runner">
    Launch the Cypress interactive test runner:
    
    ```bash
    npx cypress open
    ```
    
    This opens the Cypress launchpad where you can select E2E testing and choose your browser.
  </Step>
  
  <Step title="Select E2E Testing">
    In the Cypress interface, click on **"E2E Testing"** to configure and run end-to-end tests.
  </Step>
  
  <Step title="Choose a Browser">
    Select your preferred browser (Chrome, Edge, Firefox, or Electron) from the available options.
  </Step>
  
  <Step title="Run a Test">
    You'll see all test files from the `cypress/e2e/` directory. Click on any test file to run it:
    
    - `checkElementHome.cy.js` - Tests for home page elements
    - `historyPage.cy.js` - Tests for history page navigation
    
    The test will execute in the selected browser, and you'll see real-time results.
  </Step>
</Steps>

## Run Tests in Headless Mode

For CI/CD or faster execution, run tests in headless mode using the command line:

```bash
npm test
```

This executes the script defined in `package.json`:

```json
"scripts": {
  "test": "npx cypress run --browser chrome --headless"
}
```

<Info>
Headless mode runs tests without opening a browser window, making it ideal for automated pipelines.
</Info>

## Understanding the Configuration

The project is pre-configured in `cypress.config.js` with these settings:

### Base URL

The default base URL is set to League of Legends:

```javascript
baseUrl: 'https://www.leagueoflegends.com/'
```

<Tip>
**Customize for your project**: Update the `baseUrl` in `cypress.config.js` to point to your application.
</Tip>

```javascript
module.exports = defineConfig({
```

## Write Your First Test

Here's an example test from the project demonstrating the Screenplay Pattern:

```javascript
import * as HomeUI from '@ui/homeTarget'
import * as Quest from '@questions/elementQuest'
import * as Tasks from '@tasks/homeTask'

const NEWS = 'Noticias';

describe('Mi primera prueba en Cypress', () => {
  beforeEach(() => {
    cy.visit('es-es');
  });

  it('buscar elementos en pag lol', () => {
    // Questions - Verify element exists and is visible
    Quest.ElementExit(HomeUI.JUGAR_GRATIS);
    Quest.ElementVisible(HomeUI.JUGAR_GRATIS);
    
    // Questions - Verify image type
    Quest.CheckImageType(HomeUI.IMAGE_POPPY, 'jpg');
    
    // Questions - Verify text content
    Quest.VisibleText(HomeUI.ASSASIN_AKALY, 'La Asesina Sigilosa');
    
    // Tasks - Count elements and create aliases
    Tasks.ElementCount(HomeUI.DIVS_ELEMENT, 'Jugar gratis', 'gratisElement');
    Tasks.ElementCount(HomeUI.DIVS_ELEMENT, 'Jugar ahora', 'ahoraElement');
    
    // Questions - Verify element counts
    Quest.CheckElementNumber('ahoraElement', 3);
    Quest.CheckElementNumber('gratisElement', 2);
  });
});
```

## Using Custom Commands

The framework includes custom Cypress commands for enhanced functionality:

### Center Element and Take Evidence

```javascript
it('ejemplo con comandos personalizados', () => {
  cy.get('[data-testid="card-category"]')
    .contains('Actualizaciones del juego')
    .centerElement()                    // Scrolls element to center
    .evidence('Screenshot centrado');   // Takes labeled screenshot
});
```

### Using Environment Variables

Access environment variables in your tests:

```javascript
const secretValue = Cypress.env('SECRET_VALUE');
const variableRepo = Cypress.env('VARIABLE_REPO');

it('test with environment variables', () => {
  cy.log(`valor secreto: ${secretValue}`);
  cy.log(`variable repo: ${variableRepo}`);
});
```

<Note>
Environment variables can be set in `cypress.config.js`, `cypress.env.json`, or passed via command line.
</Note>

## Understanding the Screenplay Pattern

This project uses the Screenplay Pattern to organize test code:

<Steps>
  <Step title="UI Layer (@ui)">
    **Purpose**: Store all element selectors
    
    **Location**: `cypress/support/ui/`
    
    **Example** (`homeTarget.js`):
    ```javascript
    export const JUGAR_GRATIS = '[data-testid="play-free-button"]';
    export const IMAGE_POPPY = '.champion-image[alt="Poppy"]';
    ```
  </Step>
  
  <Step title="Tasks Layer (@tasks)">
    **Purpose**: Define user actions and business logic
    
    **Location**: `cypress/support/tasks/`
    
    **Example** (`homeTask.js`):
    ```javascript
    export const ElementCount = (elementSelector, keyWord, alias) => {
      cy.log(`Finding elements ${elementSelector} containing ${keyWord}`);
      cy.get(elementSelector)
        .filter(`:contains(${keyWord})`)
        .filter(':visible')
        .its('length')
        .as(alias);
    }
    ```
  </Step>
  
  <Step title="Questions Layer (@questions)">
    **Purpose**: Handle assertions and validations
    
    **Location**: `cypress/support/questions/`
    
    **Example** (`elementQuest.js`):
    ```javascript
    export const ElementVisible = (selector) => {
      cy.get(selector).should('be.visible');
    }
    ```
  </Step>
</Steps>

## Run Specific Tests

You can run specific test files using the `--spec` flag:

```bash
npx cypress run --spec "cypress/e2e/checkElementHome.cy.js"
```

Or run tests matching a pattern:

```bash
npx cypress run --spec "cypress/e2e/**/*home*.cy.js"
```

## Troubleshooting

### Tests failing due to timeouts

Increase the `defaultCommandTimeout` in `cypress.config.js`:

```javascript
defaultCommandTimeout: 18000  // 18 seconds
```

### Cannot find module with @ alias

Ensure webpack preprocessing is configured correctly in `cypress.config.js`:

```javascript
const webpackPreprocessor = require('@cypress/webpack-preprocessor');

setupNodeEvents(on, config) {
  const options = {
    webpackOptions: require('./webpack.config.js')
  };
  on('file:preprocessor', webpackPreprocessor(options));
}
```

### Browser not launching

Clear Cypress cache and reinstall:

```bash
npx cypress cache clear
npm install cypress --force
```
